spark入门

发表于 2017-05-16

模式：

+ 本地单机模式：所有Spark进程都运行在同一个JVM中。 
+ 集群单机模式：使用Spark自己的任务调度框架。两种模式完全兼容。
+ 基于Mesos：使用Mesos集群计算框架。
+ 基于YARN： 使用Hadoop2集群计算和资源调度框架。
  1. spark本地安装：Hadoop的文件系统HDFS；运行基于Scala（二进制包已包含scala，无需再装Scala，但需装JDK）
  2. Spark集群：集群由两类测序构成,一个驱动程序多个执行程序。本地模式时所有的处理都运行在JVM中，集群时运行在不同节点上。

    编程模型

  • SparkContext类与SparkConf类：任何Spark程序的编写都从SparkContext(使用Java编写时是JavaSparkContext)开始的。SparkContext需要初始化一个SparkConf对象，后者包含Spark集群配置的各种参数（如主节点的URL）。初始化完成后可用SparkContext的方法类创建和操作分布式数据集和共享变量。Spark shell可以自动完成上述初始化过程。Scala代码实现：
    • Spark shell: Spark 支持Scala 或 Python REPL（Read-Eval-Print-Loop,即交互式shell）。使用Scala shell：./bin/spark-shell;使用python shell./bin/pyspark.

    • 弹性分布数据集（RDD）：一个RDD代表一系列记录（某种类型的对象）。这些记录被分配到一个集群的多个节点上（本地模式下，可理解为单个进程的多个线程上）。RDD具有容错性当某个节点的任务失败时，RDD会自动在余下的节点自动重建，以便任务能最终完成。

    - Spark操作：所有操作被分为转换（Transformation，对一个数据集里的所有记录执行某种函数）和执行（action，运行某些计算和聚合操作，并将结果返回给SparkContext的驱动程序）两种。这些操作返回一个新的RDD或者计算结果。例：map操作，对一个RDD里的每一条记录都执行某个函数：

        


        注意Spark操作是延后的。在RDD上调用一个转换操作并不会立即触发相应的计算，也不会得到任何的结果。而且多个转换操作会链接起来，只有执行操作被调用时才会被高效的计算。这样大部分操作都可以在集群上并行进行。
